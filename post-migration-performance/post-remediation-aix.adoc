= Führen Sie die Host-Sanierung nach der Migration von ONTAP FLI AIX durch
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nach der Migration müssen auf den AIX-Hosts Korrekturmaßnahmen durchgeführt werden, um sicherzustellen, dass die neu importierten LUNs ordnungsgemäß erkannt werden und funktionieren.



== Nach der Migration – Offline-FLI

Für Offline-FLI bringen Sie die LUN online, nachdem der LUN-Import gelöscht und bestätigt wurde, dass die Host-Initiatoren verbunden sind.

[cols="5"]
|===
| Dateisystem | Blockgerät | LVM (FS) | Cluster | Sanboot 


| Schritt 1: Speicher auf allen zugeordneten AIX-Hosts erneut scannen | Schritt 1: Speicher auf allen zugeordneten AIX-Hosts erneut scannen. | Schritt 1: Speicher auf allen zugeordneten AIX-Hosts erneut scannen | Schritt 1: Speicher auf allen zugeordneten AIX-Hosts erneut scannen | Schritt 1: Schalten Sie den Server ein 


| Schritt 2: Identifizieren der Multipath-Geräte | Schritt 2: Identifizieren der Multipath-Geräte | Schritt 2: Identifizieren der Multipath-Geräte | Schritt 2: Identifizieren der Multipath-Geräte | Schritt 2: Überprüfen Sie, ob die Boot-Geräteliste im IPL (Initial Program Load)/SMS-Menü die neuen LUNs enthält, die auf die Geräteliste des neuen Storage-Geräts verweisen. Legen Sie die Bootreihenfolge auf den Pfad der neuen Geräteliste fest. 


| Schritt 3: Dateisystemdetails überprüfen | Schritt 3: Ermitteln Sie die Multipath-Geräte-ID und präsentieren Sie sie der jeweiligen Anwendung. | Schritt 3: Sicherstellen, dass die neuen LUNs erkannt werden | Schritt 3: Dateisystemdetails überprüfen | Schritt 3: Stellen Sie sicher, dass der Server ordnungsgemäß hochfährt 


| Schritt 4: Die Geräte wieder in die jeweiligen Ordner einbinden |  | Schritt 4: VG aktivieren | Schritt 4: Überprüfen Sie die erforderliche Clusterkonfiguration und starten Sie die Clusteranwendung | Schritt 4: Starten Sie den Host neu, um von der neu zugeordneten LUN zu booten, die Teil des LUN Import ist. 


|  |  | Schritt 5: Listen Sie die auf dem VG vorhandenen LVs auf. | Schritt 5: Überprüfen Sie den Clusterstatus und die Anwendung, die ihn verwendet. |  


|  |  | Schritt 6: Die LVMs wieder in die jeweiligen Ordner einbinden |  |  
|===

NOTE: Die aktuellste und aktualisierte Befehlsliste finden Sie in der Dokumentation des Betriebssystemherstellers.

.Schritte
. Nach dem Abtrennen erneut scannen, um den sauberen Zustand zu bestätigen.
+
[source, cli]
----
cfgmgr
----
. Speichergeräte identifizieren.
+
[source, cli]
----
lsdev -Cc disk
sanlun lun show all
sanlun lun show all -pv
lsmpio -l hdiskx
----
. Prüfen Sie, ob das Gerät angeschlossen ist.
+
[source, cli]
----
lspv |grep hdiskx
----
. Identifizieren Sie die Datenträgergruppe und die Dateisysteme.
+
[source, cli]
----
lspv
lsvg <vg_name>
lsvg -l <vg_name>
lsvg -p <vs_name>
----
. Varyonvg und Dateisysteme einbinden.
+
[source, cli]
----
varyonvg <vg_name>
mount /mount_point
df -g
----




== Nach der Migration – Online-FLI

Nach dem Löschen der Importbeziehung bestätigen Sie, dass die neue LUN unabhängig mit aktiviertem UNMAP funktioniert. Nach dem Löschen des LUN-Imports prüfen Sie, ob die LUN mit  `-space-reserve disabled` und  `-space-allocation enabled` eingestellt ist. Falls nicht, passen Sie sie entsprechend an.

[cols="4"]
|===
| Dateisystem | Blockgerät | LVM (FS) | Sanboot 


| Schritt 1: Speicher auf allen zugeordneten AIX-Hosts erneut scannen | Schritt 1: Speicher auf allen zugeordneten AIX-Hosts erneut scannen. | Schritt 1: Speicher auf allen zugeordneten AIX-Hosts erneut scannen | Schritt 1: Speicher auf allen zugeordneten AIX-Hosts erneut scannen 


|  |  | SCSI UNMAP wird automatisch aktiviert, wenn das Gerät Thin Provisioning verwendet ist. | SCSI UNMAP wird automatisch aktiviert, wenn das Gerät Thin Provisioning verwendet ist. 
|===
.Schritte
. Nach dem Abtrennen erneut scannen, um den sauberen Zustand zu bestätigen.
+
[source, cli]
----
cfgmgr
----
. Speichergeräte identifizieren.
+
[source, cli]
----
lsdev -Cc disk
sanlun lun show all
sanlun lun show all -pv
lsmpio -l hdiskx
----
. Prüfen Sie, ob das Gerät angeschlossen ist.
+
[source, cli]
----
lspv |grep hdiskx
----
. Überprüfen Sie UNMAP auf dem Gerät:
+
[source, cli]
----
chfs -a reclaim=normal /mount_point
lvmstat -v vg_name -r
----
